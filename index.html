What HTML does

The browser reads your tags like a map.

Structure

HTML organizes content into headings, paragraphs, links, images, lists, sections, and more.

Meaning

Semantic tags like <article> and <nav> help describe what each part of a page is for.

Foundation

CSS handles presentation and JavaScript handles behavior, but both depend on good HTML underneath.

Lesson 1

HTML Basics

Most HTML uses opening and closing tags. Content lives between those tags.

Elements

An element includes the tag and its content, like <p>Hello</p>.

Attributes

Attributes add extra information, like href on a link or src on an image.

Nesting

Elements can sit inside other elements, but they should still be opened and closed in the right order.

Lesson 2

Anatomy of a Web Page

A solid HTML document follows a predictable structure that browsers understand immediately. 

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <title>My First Page</title>
  </head>
  <body>
    <h1>Welcome</h1>
    <p>This is a page.</p>
  </body>
</html>

<!DOCTYPE html>

Tells the browser to use modern HTML standards.

<head>

Contains metadata, title information, and linked resources.

<body>

Contains everything the visitor actually sees on the page.

Lesson 3

Common HTML Tags

These are some of the tags learners use first and most often.

<h1> to <h6>

Headings create structure and hierarchy.

<p>

Paragraphs hold regular blocks of text.

<a>

Links connect your page to another page or location.

<img>

Images embed pictures and should include alt text.

<ul> and <ol>

Lists group related items in bullet or numbered form.

<div>

A generic container when no semantic tag fits better.

Lesson 4

Semantic HTML

Semantic elements make pages clearer for people, browsers, and assistive technology.

<header>

Introductory content for a page or section.

<nav>

A group of important navigation links.

<main>

The page's primary content, used once per page.

<section>

A thematic group of related content.

<article>

A standalone piece of content like a story or post.

<footer>

Closing content such as credits or related links.

Lesson 5

Forms and Input

Forms collect information from users. Labels are especially important for usability and accessibility.

<form>

Wraps input fields that belong together.

<label>

Describes what an input is for.

<input>

Creates fields like text boxes, email fields, and checkboxes.

<button>

Triggers submission or another action.

<form>
  <label for="email">Email</label>
  <input id="email" type="email" />

  <button type="submit">
    Subscribe
  </button>
</form>
