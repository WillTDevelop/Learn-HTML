What HTML does

The browser reads your tags like a map.

Structure

HTML organizes content into headings, paragraphs, lists, links, images, sections, and more.

Meaning

Semantic tags such as <article> and <nav> tell browsers and assistive tools what each part of a page represents.

Foundation

CSS handles presentation and JavaScript handles behavior, but HTML is the foundation both depend on.

Lesson 1

HTML Basics

Most HTML uses opening and closing tags. Content lives between them.

Elements

An element includes the tag and its content, like <p>Hello</p>.

Attributes

Attributes add extra information, like href on a link or src on an image.

Nesting

Elements can sit inside other elements. Keep them properly opened and closed to avoid broken layouts.

Lesson 2

Anatomy of a Web Page

A good HTML document follows a predictable structure that browsers expect. 

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <title>My First Page</title>
  </head>
  <body>
    <h1>Welcome</h1>
    <p>This is a page.</p>
  </body>
</html>

<!DOCTYPE html>

Tells the browser to use modern HTML standards.

<head>

Contains page metadata, title, and links to stylesheets.

<body>

Contains everything that appears on the visible page.

Lesson 3

Common HTML Tags

These tags show up in almost every project. Learn what each one is for.

<h1> to <h6>

Headings create a content hierarchy.

<p>

Paragraphs hold blocks of text.

<a>

Links connect your page to another page or location.

<img>

Images embed pictures and should include descriptive alt text.

<ul> and <ol>

Lists group related items in bullet or numbered form.

<div>

A generic container when no semantic tag fits better.

Lesson 4

Semantic HTML

Semantic elements make pages easier to understand for people, browsers, and assistive technology.

<header>

Introductory content for a page or section.

<nav>

A collection of important navigation links.

<main>

The page's primary content, used once per page.

<section>

A thematic grouping of related content.

<article>

A standalone piece of content like a post or story.

<footer>

Closing content such as credits, links, or contact info.

Lesson 5

Forms and Input

Forms collect information from users. Labels are important for usability and accessibility.

<form>

Wraps input fields that belong together.

<label>

Describes what an input is for.

<input>

Creates fields like text boxes, email fields, and checkboxes.

<button>

Triggers form submission or other actions.

<form>
  <label for="email">Email</label>
  <input id="email" type="email" />

  <button type="submit">
    Subscribe
  </button>
</form>
